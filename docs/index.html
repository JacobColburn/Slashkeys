Why SlashKeys

Private by design

Snippets are stored in your browser. No account, no cloud, no server. Nothing you type ever leaves your device.

Works everywhere you type

Gmail, Outlook web, LinkedIn, helpdesk tools, CRMs, forums — inputs, textareas, and rich editors.

The classic \trigger syntax

Same backslash triggers ShortKeys users have relied on for two decades. Instant, inline, no popup menus.

Free to use

20 snippets free, full expansion features included. One-time Pro upgrade for unlimited snippets, placeholders, and import/export — no subscription.

Switching from ShortKeys?

ShortKeys by Insight Software has been retired, and its website is gone. SlashKeys was built as its successor for the browser: the same \trigger workflow, modernized. Export your old macros as tab-delimited text and import them into SlashKeys in one click (Pro) — every trigger and replacement comes with you.

FAQ

Is it really free?
Yes — 20 snippets with full expansion features, forever. Pro is a one-time purchase (not a subscription) that unlocks unlimited snippets, dynamic placeholders like {date} and {cursor}, and import/export.
Where is my data stored?
In your browser's local extension storage, on your machine. See the privacy policy — the short version is: we never see anything.
Does it work outside the browser?
No — SlashKeys expands text in web pages only. For system-wide expansion in desktop apps, pair it with a desktop tool.
What about password fields?
Always ignored, by design.
Google Docs?
Not yet — Docs uses a canvas-based editor that doesn't expose normal text fields. It's on the roadmap.
